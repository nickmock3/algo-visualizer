# AGENTS.md

## プロジェクト概要

AlgoVis は、アルゴリズムをインタラクティブに学ぶための TanStack Start + React アプリケーションです。

各アルゴリズムページでは、ユーザーが次の3点を同時に理解できる状態を目指します。

- そのアルゴリズムがどの問題を解くのか。
- アルゴリズムの状態がステップごとにどう変化するのか。
- 現在のステップが Rust コードのどの行に対応するのか。

## 開発コマンド

既存の npm scripts を使います。

```sh
npm run dev
npm run build
```

変更完了前には `npm run build` を実行してください。本番ビルドと TypeScript チェックを兼ねています。

## コード配置

- アルゴリズムのステップ生成ロジックは `src/algorithms/*.ts` に置きます。
- ルート単位の UI は `src/routes/algorithms/*.tsx` に置きます。
- 共通スタイルは `src/styles/app.css` に置きます。
- `src/routeTree.gen.ts` は TanStack Router の生成ファイルです。手で編集せず、dev server または build に更新させます。

アルゴリズムのロジックは React コンポーネントから分離してください。React ページは、イベントハンドラや render 内にアルゴリズム本体を埋め込むのではなく、事前に生成した step オブジェクトを表示する構成にします。

## アルゴリズムページの基本形

新しいアルゴリズムページは、既存の visualizer パターンに合わせます。

- 例題データを入力するコントロール。
- 再生、一時停止、戻る、次へ、リセット、速度調整のステップ操作。
- 現在状態のビジュアル表示。
- 短い説明を表示する「現在のステップ」パネル。
- アクティブ行をハイライトする Rust コードパネル。
- 必要に応じた計算量や判定条件のパネル。
- 長めの概念説明を置く独立した説明セクション。

アルゴリズムコードの例は、明示的に別言語を求められていない限り Rust を優先します。

## UI とレイアウト

レイアウト規約は [docs/layout-guidelines.md](docs/layout-guidelines.md) にまとめています。

UI を追加・変更する場合は、明示的に別スタイルを求められていない限り既存の見た目に合わせます。現在の基準は `lower_bound` ページのスタイルです。明るいサーフェス、インディゴのアクセント、コンパクトな角丸パネル、控えめな影を使います。

## 実装方針

- 変更範囲は、依頼されたアルゴリズムまたはそのために必要な共通ヘルパーに絞ります。
- 可視化ステップには、小さく型付けされたデータ構造を使います。
- ステップ説明の文言は、日本語で短く読みやすくします。
- オーバーフローが起こり得る例では、整数として安全な式を使います。
- 基本的な可視化のために新しい依存関係を追加することは避けます。
- 新しい抽象化を作る前に、近くの route ファイルにある既存パターンを優先します。

## 検証

アルゴリズムページを変更した場合は、次を確認します。

- `npm run build` を実行する。
- `npm run dev` を起動、または既存の dev server を再利用する。
- 対象ルートがローカルで応答することを確認する。

例:

```sh
curl -I http://localhost:3000/algorithms/newton-sqrt
```

レイアウト、レスポンシブ挙動、パネルサイズを変更した場合は、可能な限りブラウザで表示を確認してください。
