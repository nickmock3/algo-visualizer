# レイアウト規約

## ページ構成

アルゴリズムページは、縦方向に2つのセクションとして読める構成にします。

1. インタラクティブな可視化セクション。
2. 概念を説明するセクション。

1つ目のセクションは、操作しながら学ぶためのメイン画面です。説明が短いヒントではなく長めの文章になる場合は、2つ目のセクションとして次のビューポートに配置します。

## 可視化セクションのレイアウト

横幅が広い画面では2カラム構成を使います。

- 左カラム: 入力、アニメーションまたは可視化、再生操作、現在のステップ。
- 右カラム: Rust コード、計算量、アルゴリズム固有のメモ。

左右のカラムは、視覚的な高さと情報量のバランスを保ちます。片側だけに長い説明文が入り、作業画面の高さが大きく崩れる場合は、その説明を独立した説明セクションへ移動します。

狭い画面では縦に積み、可視化をコードより先に表示します。

## 説明セクション

長めの概念説明は、横幅いっぱいに複数カラムで並べないでください。読みやすい1本の本文カラムを優先します。

- デスクトップでは半ページ程度の幅にし、目安は `max-w-[680px]` から `max-w-[760px]` です。
- 段落は縦に積みます。
- 2ページ目として読めるよう、1ビューポートに近い縦方向の余白を持たせます。
- 見出しは短く明確にします。

短いヒントであれば、可視化セクション内の小さなパネルに残して構いません。

## 見た目の基準

明示的に別テーマを求められていない限り、現在の `lower_bound` スタイルに合わせます。

- 背景: 明るいインディゴ寄りのページ背景。
- アクセント: バッジ、ボタン、フォーカスリング、スライダー、アクティブ表示にはインディゴを使います。
- パネル: 白の半透明サーフェス、控えめな境界線、柔らかい影を使います。
- 角丸: `rounded-lg` または `rounded-xl` 程度のコンパクトな角丸を使います。
- 文字色: slate 系を使い、見出しは強く、本文は読みやすくします。

ページごとに独自のカラーテーマを作ることは避けます。アルゴリズムページ間の一貫性を、見た目の新規性より優先します。

## 操作コントロール

再生操作は、可視化の近くに置きます。

- 主操作: 再生、一時停止。
- 補助操作: 戻る、次へ、リセット。
- 速度調整: スライダー。

ボタンには、無効状態とフォーカスリングを必ず用意します。

## コードパネル

Rust コードパネルは次の形式にします。

- 暗い背景。
- 等幅フォント。
- 行番号。
- 現在ステップに応じたアクティブ行のハイライト。

コード例は、可視化を見ながら追える長さに抑えます。

## レスポンシブ対応

モバイルでも操作できる状態を保ちます。

- 密なグリッドは縦積みにします。
- はみ出す固定幅を避けます。
- コードブロックや横長の可視化行は、必要に応じて横スクロール可能にします。
- ラベルは常に表示し、プレースホルダーに依存しません。

テキストがマーカー、操作ボタン、パネルと重ならないようにします。再生中にページが大きく揺れないよう、可視化要素には安定した寸法を持たせます。
